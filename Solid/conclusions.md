### Принцип единственной ответственности

`NumberGenerateService` - Генерация случайных чисел в диапазоне
`DecisionNumberMakeService` - Сравнение предположения с целью, подсчёт попыток
`OutputService` - Форматирование вывода в консоль
`StatusGameService` - Оценка условий победы/поражения

### Принцип открытости/закрытости
`BaseOutputService` предоставляет шаблон для сервисов вывода, его наследники изменяют формат вывода

### Принцип подстановки Барбары Лисков (Liskov Substitution Principle)

Объекты подкласса должны быть взаимозаменяемыыми с объектами класса - родителя без поломки программы. В данном случае, используя  FunnyOutputService мы 
не ломаем программу. Классы наследники переопределяют настройки вывода и сообщения, сам метод вывода остаётся один на всех, т.е. мы не меняем контракт

### Принцип разделения интерфейса (Interface Segregation)

`INumberGenerateService` - только генерация чисел
`IStatusGameService` - только оценка состояния игры
`IDecisionNumberMakeService` - только сравнение чисел + подсчёт попыток
`IOutputService`- только операции вывода

### Принцип инверсии зависимостей
Все зависимости внедряются через интерфейсы, Program зависит от абстракций**, а не от конкретных классов, DI-контейнер настроен в `DependencyInjection.cs`
